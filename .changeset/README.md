# Changesets

Hi! This folder has been automatically generated by `@changesets/cli`, a build tool that works with multi-package repos, or single-package repos to help you version and publish your code. You can find the full documentation at https://github.com/changesets/changesets.

## Adding a changeset

Run `bun run changeset` (or `bunx changeset`) — an interactive CLI that asks:

1. **Which packages changed?** (only one here — `ultrareview-clone`)
2. **Is this a major / minor / patch change?**
   - **patch** — bug fix, doc tweak, internal refactor
   - **minor** — new feature, backward compatible
   - **major** — breaking change (API, CLI, config, action inputs)
3. **Summary** — 1-2 sentence description that lands in CHANGELOG.md

This creates `.changeset/<random-slug>.md`. Commit it with your code changes.

## Release flow (automated)

1. PR merged to `main` with changesets
2. `Release` workflow opens a "Version Packages" PR bumping `package.json` + regenerating `CHANGELOG.md`
3. Maintainer merges the Version PR → workflow creates a git tag + GitHub Release

See `.github/workflows/release.yml` for details.

## Do NOT manually edit

- `CHANGELOG.md` — generated from changesets
- `package.json` version — bumped by changesets

## What to edit

- `.changeset/*.md` — your changeset entries
- `.changeset/config.json` — tool config (change with care)
